=== ЗАДАЧА (Step 5 — контролно качество) ===
Прегледай всяко хранене и оправи проблемите. Запази type и dessert без промяна.

Можеш и трябва да променяш при нужда:
• description — продукти + грамажи (формат "• продукт 150g" на всеки ред)
• name — ясно, кратко име на ястието
• benefits — 1 изречение здравословен ефект

Какво проверяваш и оправяш:
1. Неадекватни комбинации (несъвместими продукти, дублирани източници на въглехидрати, липса на основен белтъчен източник)
2. Абсурдни грамажи (твърде малки/големи, подправки като основен макроизточник, нереалистични порции)
3. Липсващи или неправилни g/г в description
4. name/benefits — по-ясни и естествени на български

НЕ правиш:
• НЕ променяй type или dessert
• НЕ връщай calories, macros или weight — бекендът ги преизчислява от продуктите след този стъпка
• НЕ добавяй dailyTotals
• НЕ измисляй продукти, които нарушават алергии/blacklist

При "Свободно хранене": запази type; benefits — насърчителен тон.

JSON ФОРМАТ (дни {startDay}-{endDay}):
{
  "day{startDay}": {
    "meals": [
      {"type": "...", "name": "...", "description": "• продукт 150g\n• ...", "benefits": "..."}
    ]
  }
}
Върни САМО валиден JSON. Копирай type и dessert без промяна. Коригирай description, name и benefits където е нужно.
