# Настройка на Make.com Webhook → Telegram бот

## Как работи сега

Системата изпраща до Make.com JSON с **едно поле** – `telegramMessage` – което вече съдържа готовото, форматирано съобщение за Telegram. Датата се показва в четим формат (напр. `22.04.2026 в 22:54`), а всеки тип събитие включва само своите полета.

---

## Примерни съобщения

**Ново запитване:**
```
📩 Ново запитване

👤 Клиент: Иван Петров
🆔 ID: client_1234abc
📧 Имейл: ivan@example.com
🎯 Цел: Отслабване
📅 Дата: 22.04.2026 в 22:54
```

**Нов план чака преглед:**
```
📋 Нов план чака преглед

👤 Клиент: Иван Петров
🆔 ID: client_1234abc
📧 Имейл: ivan@example.com
🎯 Цел: Отслабване
📅 Дата: 22.04.2026 в 22:54
```

**Доклад за проблем:**
```
🐛 Доклад за проблем

👤 Потребител: Иван Петров
🆔 ID: ivan@example.com
💬 Съобщение: Не работи менюто
📅 Дата: 22.04.2026 в 22:54
```

## Настройка в Make.com

### 1. Накарайте webhook модула да „преоткрие" структурата

1. Отворете сценария в [make.com](https://make.com).
2. Кликнете върху **Webhook** модула.
3. Натиснете **"Redetermine data structure"** и изчакайте следваща заявка от платформата.
4. Make.com ще разпознае полето `telegramMessage`.

### 2. Актуализирайте Telegram модула

1. Кликнете върху **Telegram** модула.
2. В полето **"Text"** поставете само:

```
{{1.telegramMessage}}
```

> `{{1.telegramMessage}}` означава „поле `telegramMessage` от модул 1 (webhook)".  
> Ако вашият webhook е друг номер (напр. 3), използвайте `{{3.telegramMessage}}`.

3. Кликнете **OK** и запазете сценария.

### 3. Проверете дали сценарият е активен

Уверете се, че превключвателят в горния десен ъгъл е в позиция **ON**.

## Резюме на промяната в кода

**Преди:** Изпращаха се много отделни полета (`name`, `userId`, `subject`, `clientId`, `email`, `goal`, `reportMessage`, `timestamp`). Telegram шаблонът ги показваше всички, включително празните.

**Сега:** Изпраща се едно поле `telegramMessage` с готовото, форматирано съобщение. Всеки тип събитие включва само своите полета, а датата е в четим формат.
