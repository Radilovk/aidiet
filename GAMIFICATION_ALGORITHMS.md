## 1. Структура на данните (`rec` обект)

Всеки ден се пази в `localStorage` под ключ `gameData` → `{ "YYYY-MM-DD": rec, ... }`.

```js
rec = {
  date:            "YYYY-MM-DD",
  meals:           { "Закуска": false, "Обяд": true, "Вечеря_2": false, ... },
  mealCalories:    { "Закуска": 380, "Обяд": 520, ... },     // kcal при отбелязване
  extraMeals:      [ { name, calories, isJunk, isFreeMealReplacement, suitabilityScore, ts }, ... ],
  freeMealRatings: { "Свободно хранене": 4, ... },            // оценка 1–5 от AI анализ
  morningCheck:    { sleptWell: true|false, ts },
  eveningCheck:    { activityLevel: 1–3, emotionalBalance: 1–3, waterIntake: true|false, ts },
  plannedCalories: 1640,    // сума kcal от план (четена от DOM)
  dailyScore:      { score, stars, label, ... },
  missing:         false
}
```

**Ключове за хранения:**
- Първото хранене от дадения тип → `"Вечеря"`
- Второто/следващо хранене от същия тип → `"Вечеря_2"`, `"Вечеря_3"`, …
- Добавени от снимка хранения (`.added-meal-card`) **не** се включват в `rec.meals`  
  (вече са в `rec.extraMeals` чрез `logExtraFood`).

## 2. Алгоритъм за дневна оценка (`calcDayScore`)

### 2.1 Точки от хранения
| Условие | Точки |
|---------|-------|
| Планирано хранене отбелязано като готово | +10 |
| Свободно хранене — оценка AI ≥ 4/5 | +10 |
| Свободно хранене — оценка AI < 4/5 | 0 + флаг `incorrectMeals` |
| Свободно хранене — без снимка (неоценено) | 0 (не се наказва) |

`mealMax = брой хранения × 10`

> **Важно:** При наличие на суфикс (`"Свободно хранене_2"`), базовият тип се извлича чрез `m.replace(/_\d+$/, '')` преди да се провери дали е Свободно хранене.

### 2.2 Wellness точки (max 40)
| Критерий | Точки |
|----------|-------|
| Добър сън (`morningCheck.sleptWell = true`) | 10 |
| Вода ≥ 2 л (`eveningCheck.waterIntake = true`) | 10 |
| Ниво активност 3/3 | 10 |
| Ниво активност 2/3 | 5 |
| Ниво активност 1/3 | 0 |
| Емоционален баланс 3/3 | 10 |
| Емоционален баланс 2/3 | 5 |
| Емоционален баланс 1/3 | 0 |

`wellnessMax = 40` (знаменателят включва wellness дори при непопълнени въпроси — гарантира максимум 5★ само при 100% от всичко)

### 2.3 Звезди
```
pct = totalEarned / totalMax   (totalMax = mealMax + 40)

pct ≥ 100% И без блокери → 5 ★  (Отличен ден!)
pct ≥  80%               → 4 ★  (Много добре!)
pct ≥  55%               → 3 ★  (Добре)
pct ≥  30%               → 2 ★  (Може по-добре)
pct ≥   1%               → 1 ★  (Подобри се утре)
pct =   0% или нула      → null  (Без данни)
```

### 2.4 Блокери за 5 ★
Всяко от следните предотвратява 5★ (максимумът пада до 4★):
- Не всички планирани хранения отбелязани
- Неправилно свободно хранене (оценка < 4/5)
- Калориен излишък > 10% над плана
- Лош сън (`sleptWell = false`)
- Ниска вода (`waterIntake = false`)
- Ниска активност (ниво 1)
- Нисък баланс (ниво 1)
- Вредни храни (`junkCount > 0`)

## 3. Калориен баланс

```
completedCals = Σ mealCalories[m]  за всяко m с rec.meals[m] === true
extraCals     = Σ em.calories       за em.isFreeMealReplacement === false в extraMeals
totalConsumed = completedCals + extraCals

excessPct = (totalConsumed - planned) / planned
```

| `excessPct` | Статус |
|-------------|--------|
| > 10% | `surplus` + флаг `excessCalories = true` (блокира 5★) |
| 0–10% | `surplus` (предупреждение, не блокира) |
| < −10% и ден е приключил | `deficit` |
| иначе | `balanced` |

---

## 4. Ангажираност (engPct)

Запазен за обратна съвместимост (показва се в седмичния recap).

```
mealEngPct = (done / mealCount) × 50     (max 50 pts)
mornEngPct = 15 ако morningCheck е попълнен
eveEngPct  = 15 ако eveningCheck е попълнен
junkPct    = max(0, 20 − (junkCount + incorrectMeals) × 7)

engPct = mealEngPct + mornEngPct + eveEngPct + junkPct   (0–100)
```

## 5. Отключване на бутоните за отбелязване (Time Lock)

| Тип хранене | Достъпно след |
|-------------|--------------|
| Закуска | 06:00 |
| Обяд | 12:00 |
| Следобедна закуска / Десерт | 14:00 |
| Вечеря / Свободно хранене | 18:00 |
| Всички останали | 06:00 |

- Правилото важи само за **текущия ден**. За минали дни — всички бутони са активни.
- Интервал от 60 секунди периодично проверява и отключва бутони при достигане на часа.

---

## 6. Седмична оценка (`calcWeeklyScore`)

```
avgScore   = среден брой звезди за последните 7 дни (0–5)
avgEngPct  = средна ангажираност % за 7 дни
noDataDays = дни без никакви данни (не попълнени въпроси/хранения)
```

Дни без данни се броят като 0% ангажираност и 0 звезди.

## 7. Анимация при спечелване на звезди (`triggerStarCelebration`)

Анимацията се активира **само когато броят звезди се увеличава** (не при зареждане на страницата).

| Ниво | Анимация на картата | Летящи елементи | Хаптика | Допълнително |
|------|---------------------|-----------------|---------|--------------|
| ★ | Лек pop | 6 | 60 ms | — |
| ★★ | Shake | 10 | [60,40,80] ms | — |
| ★★★ | Bounce | 16 | [80,40,80,40,120] ms | — |
| ★★★★ | Glow 4 | 24 | [100,50,100,50,150,50,150] ms | — |
| ★★★★★ | Glow 5 (максимален) | 36 (по-големи) | [150,60,...,300] ms | Flash банер "Отличен ден!" |

## 8. Хаптична обратна връзка

| Събитие | Вибрация |
|---------|----------|
| AI асистент изпраща съобщение | 120 ms buzz + стаггиран per-символ паттерн |
| Геймификация typewriter (нов символ) | 1 ms vibrate на всеки 3 символа |
| Спечелена звезда | виж Таблица 7 |

Всички вибрации ползват `navigator.vibrate()` и работят само на устройства, поддържащи Vibration API (Android Chrome; iOS игнорира без специална настройка).

## 9. Корелации (аналитичен модул `analysis.html`)

Скорингът в `analysis.html` е **отделен** от геймификацията и е базиран на AI-предоставени данни:

- `analysis.currentHealthStatus.score` (0–100) — текущо здравословно състояние от AI
- 10% корекция надолу е вградена в backend промпта (не в frontend-а)
- Fallback: `score = 100 − avg(severityValues)` при липса на `currentHealthStatus`
- Минимум: 15 (човекът е жив дори при критично здраве)
