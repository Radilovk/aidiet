# AI Логове

Тази папка се обновява автоматично след всяко успешно генериране на хранителен план.

## Структура

```
ai-logs/
└── latest/                     ← последното генериране
    ├── communication_log.txt   ← пълен лог: промпт + отговор за всяка стъпка
    ├── ai_settings.json        ← модел, провайдър и параметри (temperature, topP …)
    └── final_plan.json         ← сглобеният хранителен план (без лични данни)
```

## Съдържание на файловете

### `communication_log.txt`
Пълен запис на комуникацията между бекенда и AI модела:
- Стъпка 1 – Анализ на профила (`step1_analysis`)
- Стъпка 2 – Диетична стратегия (`step2_strategy`)
- Стъпка 3 – Хранителен план (`step3_meal_plan_*`)
- Стъпка 4 – Финално резюме (`step4_summary`)

За всяка стъпка са записани: параметри, входен профил (без име/email/телефон), бекенд калкулации, пълния промпт и пълния AI отговор.

### `ai_settings.json`
Настройките на AI модела по времето на генерирането:
- `plan` – провайдър, модел, temperature, topP, topK, thinking budget, token limits по стъпки
- `chat` – настройки за чат асистента
- `vision` – настройки за анализ на снимки на храна

### `final_plan.json`
Сглобеният 7-дневен хранителен план, точно такъв какъвто е изпратен на клиента.

## Изисквания

За да работи автоматичното записване трябва да е конфигуриран Cloudflare Worker secret:
```
wrangler secret put GITHUB_TOKEN
```
Стойността трябва да е GitHub Personal Access Token (PAT) с `repo` scope (write access).

По подразбиране се записва в бранч `main` на репото `Radilovk/aidiet`.  
Може да се промени с optional secrets `GITHUB_REPO` и `GITHUB_BRANCH`.
